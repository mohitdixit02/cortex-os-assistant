# Database Schema

## Global Enums
RoleType: USER, AI
AIClientType: VOICE_CLIENT, CORE_MAIN_CLIENT
PreferenceLevel: MUST, SHOULD, CAN, CANNOT
TimeOfDay: MORNING, AFTERNOON, EVENING, NIGHT
MoodTrend: STABLE, INCREASING, DECREASING

### Users Table (Google OAuth)
user_id: UUID (PK)
google_id: String (Unique, Indexed) — From Google OAuth sub field
email: String (Unique, Indexed)
full_name: String
phone_number: String (Nullable)
profile_picture: String (URL)
created_at: Timestamp
updated_at: Timestamp
deleted_at: Timestamp (Nullable)

### ChatSessions (The "Thread")
session_id: UUID (PK)
user_id: UUID (FK -> Users.user_id)
summary: String (Nullable) — Generated by STM later
created_at: Timestamp
updated_at: Timestamp

### Messages (Individual Turns)
message_id: UUID (PK)
session_id: UUID (FK -> ChatSessions.session_id)
user_id: UUID (FK -> Users.user_id)
content: Text
role: Enum(RoleType)
ai_client: Enum(AIClientType) (Nullable)
is_summarized: Boolean (Default: False)
is_tool_used: Boolean (Default: False)
tool_id: String (Nullable)
embedding: Vector — For pgvector search (variable dimensions)
created_at: Timestamp

### UserShortTermMemory (STM)
stm_id: UUID (PK)
user_id: UUID (FK -> Users.user_id)
session_id: UUID (FK -> ChatSessions.session_id)
stm_summary: Text
session_preferences: JSON/Text — Based on this specific chat
created_at: Timestamp
updated_at: Timestamp

### UserEmotionalProfiles (LTM: Mood & Behavior)
profile_id: UUID (PK)
user_id: UUID (FK -> Users.user_id)
session_id: UUID (FK -> ChatSessions.session_id)
mood_type: String (e.g., "Happy", "Sad")
time_behavior: Enum(TimeOfDay)
emotional_level: Integer (Constraint: 1-10)
logical_level: Integer (Constraint: 1-10)
social_level: Integer (Constraint: 1-10)
context_summary: Text — Constraint specific preferences (used to tune the reply based on current emotional and time state)
created_at: Timestamp

### UserKnowledgeBase (LTM: Preferences & Traits)
trait_id: UUID (PK)
user_id: UUID (FK -> Users.user_id)
strictness: Enum(PreferenceLevel) (Nullable)
content: Text — e.g., "Does not like being interrupted during work"
is_active: Boolean (Default: True)
embedding: Vector — Used for finding relevant preferences during a query (variable dimensions)
created_at: Timestamp
updated_at: Timestamp

### Tools Table (The Registry)
tool_id: UUID (PK)
tool_name: String (Unique, e.g., "send_whatsapp")
tool_description: Text — The actual prompt description the LLM reads to understand how to use it.
is_active: Boolean (Default: True)
created_at: Timestamp
updated_at: Timestamp
deleted_at: Timestamp (Nullable for Soft Delete)

### Tasks Table (The Queue)
task_id: UUID (PK)
message_id: UUID (FK -> Messages.message_id) — Links the task to the specific user request.
tool_id: UUID (FK -> Tools.tool_id, Nullable)
task_name: String
task_description: Text (Nullable) — Canonical description used for embedding generation and retrieval.
status: Enum(INITIALIZED, QUEUED, PROCESSING, COMPLETED, FAILED)
payload: JSON — The input arguments (e.g., {"number": "123", "text": "Hi"}).
status_response: JSON/Text — The output or error message from the tool.
metadata: JSON — Extra info like 'retry_count' or 'execution_time_ms'.
embedding: Vector (Nullable) — Vector embedding of task_description for semantic search.
created_at: Timestamp
updated_at: Timestamp
